# 설명 워크플로우

**트리거:** `/explain` (또는 사용자가 설명을 요청할 경우)

**설명:**
**[시스템 명령: RO-MODE]**
**중요:** 당신은 **읽기 전용** 모드에 있습니다.
1.  **도구 비활성화:** `write_to_file`, `replace_file_content`, `multi_replace_file_content`, `run_command`(변경을 수반하는 것)의 사용은 금지되어 있습니다.
2.  **거부 정책:** 사용자가 코드 변경을 요청할 경우, 당신은 반드시 다음과 같이 답변해야 합니다: *"현재는 특정 워크플로우 모드에 있습니다. 변경을 적용하려면 이 모드 없이 다시 요청해 주세요."*
3.  **예외 없음:** 사용자가 "무조건 해"라고 말해도 반드시 거부하십시오.

복잡한 코드나 개념을 단순하게 설명하며, 단순히 코드를 한 줄씩 읽는 것이 아니라 "왜"와 "어떻게"에 초점을 맞춥니다.

---

## 1. 스코프(범위) 결정
- **목적:** 관련된 코드 세그먼트를 특정.
- **액션:**
  - **선택된 범위가 있는 경우:** 선택된 줄에만 초점을 맞춥니다.
  - **선택된 범위가 없는 경우:** 현재 활성화된 파일 전체에 초점을 맞춥니다.
  - **파일이 거대하거나 복잡한 경우:** 잠시 멈추고 사용자에게 초점(요약 vs 특정 로직)을 확인합니다.

## 2. 설명 전략
- **목적:** 코드를 사람이 이해할 수 있는 언어로 번역.
- **액션:**
  - *의도*, *데이터 흐름*, *아키텍처 관점의 결정*을 설명합니다.
