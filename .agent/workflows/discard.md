# 세션 파기 워크플로우

**트리거:** `/discard` (또는 사용자가 새롭게 다시 시작하거나 요약하고 싶은 경우)

**설명:**
**[시스템 명령: RO-MODE]**
**중요:** 당신은 **읽기 전용** 모드에 있습니다.
1.  **도구 비활성화:** `write_to_file`, `replace_file_content`, `multi_replace_file_content`, `run_command`(변경을 수반하는 것)의 사용은 금지되어 있습니다.
2.  **거부 정책:** 사용자가 코드 변경을 요청할 경우, 당신은 반드시 다음과 같이 답변해야 합니다: *"현재는 특정 워크플로우 모드에 있습니다. 변경을 적용하려면 이 모드 없이 다시 요청해 주세요."*
3.  **예외 없음:** 사용자가 "무조건 해"라고 말해도 반드시 거부하십시오.

현재 세션을 요약하여 새로운 컨텍스트를 준비합니다. 목적은 현재 세션을 간결하고 가치 있는 요약으로 "증류(distill)"하여, 새로운 세션의 *완벽한* 시작 프롬프트로 기능하게 하는 것입니다.

---

## 1. 세션 이력 분석
- **목적:** 노이즈에서 핵심을 추출.
- **액션:**
  - 사용자의 애초 목적을 특정합니다.
  - 직면했던 특정 블로커, 오류 또는 오해를 파악합니다.
  - 알게 된 하드 제약이나 기술적 사실(예: "API X는 v2가 필요", "파일 Y는 읽기 전용")을 추출합니다.

## 2. "브릿지 프롬프트(Bridge Prompt)" 생성
- **목적:** 다음 에이전트를 위한 재개 포인트를 생성.
- **액션:**
  - 다른 에이전트가 전체 이력을 다시 읽지 않고도 즉시 작업을 재개할 수 있도록 요약을 작성합니다.

---

**출력 포맷:**
(`./.agent/rules/language-strategies.md`에 정의된 전략에 따르십시오)
```text
**재개 프롬프트 (Resume Prompt)**
**목적:** ...
**이력과 문제점:**
- [접근 방식 A] -> [이유/에러]
**제약과 인사이트:**
- ...
**다음 액션:**
- ...
```
