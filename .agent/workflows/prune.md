# Git Branch Prune Workflow

**Activation:** 이 워크플로우는 사용자가 `/prune` 명령어를 사용하거나 원격 저장소에서 이미 삭제된 로컬 브랜치들의 일괄 정리를 요청했을 때 활성화됩니다.

## Objective
원격 저장소(Remote)에서는 병합되거나 삭제되었지만, 로컬 환경에는 여전히 남아있는 쓸모없는 브랜치(`[gone]` 상태)들을 찾아내어 일괄 삭제함으로써 Git 브랜치 트리를 깨끗하게 유지합니다.

## Workflow Steps

### 1. 메인 브랜치 이동 및 최신화 (`checkout` & `pull`)
- 내장된 **`run_command`** 도구를 사용하여 다음 명령어를 실행합니다.
  ```bash
  git checkout main && git pull
  ```
- 브랜치를 정리하기 전에 기준이 되는 `main` 브랜치로 이동하여 원격의 최신 변경사항을 반영합니다.

### 2. 동기화 및 고립된 브랜치 추적 (`fetch --prune`)
- 내장된 **`run_command`** 도구를 사용하여 다음 명령어를 실행합니다.
  ```bash
  git fetch --prune
  ```
- 이 명령어는 원격 저장소에서 삭제된 브랜치의 로컬 추적(Tracking) 정보를 정리합니다.

### 3. 삭제 대상 브랜치 식별 (`branch -vv`)
- 내장된 **`run_command`** 도구를 사용하여 다음 명령어를 실행합니다.
  ```bash
  git branch -vv
  ```
- 명령어의 출력 결과를 분석하여 `[gone]` 키워드가 포함된 브랜치들의 목록을 추출합니다.
- `main` 브랜치는 절대 삭제 대상에 포함되지 않아야 합니다. (일반적으로 `[gone]` 상태가 되지 않지만 안전 장치로 간주)

### 4. 브랜치 일괄 삭제 (`branch -D`)
- 추출된 `[gone]` 상태의 브랜치 목록이 **있는 경우**:
  - 사용자에게 "다음 브랜치들이 원격에서 삭제된 것으로 확인되어 로컬에서 정리를 진행합니다"라고 목록을 안내합니다.
  - 내장된 **`run_command`** 도구를 사용하여 해당 브랜치들을 강제 삭제(`-D`)합니다.
    ```bash
    git branch -D <branch-name-1> <branch-name-2> ...
    ```
  - 삭제 결과를 사용자에게 보고합니다.

- 추출된 `[gone]` 상태의 브랜치 목록이 **없는 경우**:
  - "로컬에 정리할 고립된 브랜치가 없습니다. 브랜치 트리가 깨끗합니다!"라고 보고하고 워크플로우를 종료합니다.
