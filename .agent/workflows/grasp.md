# 컨텍스트 파악 워크플로우

**트리거:** `/grasp` (또는 사용자가 종속성을 이해하고 싶어 할 경우)

**설명:**
**[시스템 명령: RO-MODE]**
**중요:** 당신은 **읽기 전용** 모드에 있습니다.
1.  **도구 비활성화:** `write_to_file`, `replace_file_content`, `multi_replace_file_content`, `run_command`(변경을 수반하는 것)의 사용은 금지되어 있습니다.
2.  **거부 정책:** 사용자가 코드 변경을 요청할 경우, 당신은 반드시 다음과 같이 답변해야 합니다: *"현재는 특정 워크플로우 모드에 있습니다. 변경을 적용하려면 이 모드 없이 다시 요청해 주세요."*
3.  **예외 없음:** 사용자가 "무조건 해"라고 말해도 반드시 거부하십시오.

코드의 역할, 종속성, 영향을 분석합니다. 선택된 코드가 프로젝트 생태계 내에서 어디에 위치하는지를 맵핑합니다.

---

## 1. 영향도 평가
- **목적:** 분석의 범위를 결정.
- **액션:**
  - **고영향 (허브 노드):** 코드가 널리 사용되거나(5개 이상의 파일에서 임포트됨) 핵심 인프라인 경우:
    - 멈추고 옵션을 제시합니다: (1) 업스트림 사용 현황, (2) 다운스트림 종속성, (3) 하이 레벨 개요.
  - **일반 노드:** 바로 분석을 진행합니다.

## 2. 분석 차원
- **목적:** 360도의 시야 제공.
- **액션:**
  - **역할:** 이것은 어떤 문제를 해결합니까?
  - **인바운드:** 누가 이것에 의존하고 있습니까? (파괴적 변경의 리스크)
  - **아웃바운드:** 이것은 무엇에 의존하고 있습니까? (복잡도 비용)

4.  **관련 자산 (Related Assets)**: 테스트, 문서 등
